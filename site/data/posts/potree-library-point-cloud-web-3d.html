One of the fastest-growing applications in 3D is scanning objects and spaces and converting them to digital form.

When a scan is processed — whether captured by photography, laser, or another method — it is translated into a collection of colored points in space called a Point Cloud. Based on this collection of points, various software programs and applications can translate them back into a virtual object that can be moved, rotated, and even navigated through.

Each point in a Point Cloud is assigned X, Y, and Z position values as well as an RGB color value. In most cases, the point cloud is built from a sequence of images. A common use case for point clouds arises when a drone is deployed to scan a tourism site, construction site, or any other terrain area. The drone photographs the area from multiple angles, and based on each image's angle and position data, the cloud is constructed. In a similar way, rooms, vehicles, aircraft, or any other large object can be scanned using this method.

The Performance Problem: Handling a Cloud of Millions of Points in Space

When large spaces or objects are scanned, an enormous point cloud is created that can reach billions of points. In most cases, the resulting cloud is too large to be stored in the memory and computing power of a phone or even a regular computer, let alone downloaded over the internet. To enable the display of the object — the area that was photographed and scanned — the data (the point cloud) must be organized in a memory-efficient method and in a specific order that allows for fast retrieval and editing.

Potree Is Up to the Task

Potree is a library developed specifically to display large point clouds using an advanced, memory-efficient method, particularly for the Web. Potree's great advantage is its ability to show users only the information relevant to them at any given moment, thereby reducing the number of points to be processed and the computing power required. In fact, Potree is currently the only tool available for efficiently displaying such rich data on the Web.

According to Potree's algorithm, when viewing from a distance, the minimum number of points necessary to make the model image appear "smooth" and "complete" will be displayed, even though it won't contain all the points. As you zoom in, more and more points are revealed. Other points that are currently outside your field of view are removed to avoid overloading the computer's rendering engine. This is somewhat similar to how Google Maps works.

Potree, built as an open-source library based on WebGL, is supported primarily by the company Rapidlasso and several additional contributors. Unlike many libraries on the market that have accessible and clear APIs, Potree is not one of them. To achieve good performance and make the most of its capabilities for processing and displaying point clouds, deep knowledge of 3D graphics in general and the THREE.js engine in particular is required.

With this kind of expertise, it is possible to enrich this environment and tailor it to the unique needs of each application. For example: integrating models in a 3D environment, high-precision area and volume measurements, overlay on maps, graphical display of data (such as elevation or heat measurements) by color, and comparison of changes over time.

In summary, when tackling the challenge of displaying 3D and data clouds on the web, Potree provides the most efficient approach — one that supports cross-platform requirements and hybrid Point Cloud display on any device.
